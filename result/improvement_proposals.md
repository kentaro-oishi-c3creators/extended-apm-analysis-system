# 拡張APM分析システム 改善提案書

現状のシステムをより使いやすく、実践的なツールとして進化させるための改善案をまとめました。

## 1. UX・データ分析の強化

### 1-A. レーダーチャート（蜘蛛の巣グラフ）の追加
- **課題**: 現在は「テーブル」と「散布図（インパクト×努力）」のみで、個々のプロジェクトの強み・弱み（ワクワク度が高いのか、精神摩耗が低いのかなど）を直感的に把握しづらい。
- **提案**: `recharts` の `RadarChart` を用いて、5つの評価軸（インパクト、努力、精神摩耗、ワクワク度、凸性）のバランスを可視化する。
- **メリット**: 「この案件はワクワク度は高いが努力も大きい」といった特性が一目でわかるようになる。

### 1-B. パーソナリティ設定のプリセット化（UI強化）
- **課題**: 現在、パーソナリティは自由記述のテキストボックスであり、ユーザーが毎回考える必要がある。
- **提案**: 「分析・論理重視」「直感・パッション重視」「リスク回避型」などのプリセットボタンを用意し、ワンクリックで設定できるようにする（自由記述も残す）。
- **メリット**: 入力の手間が省け、AI（またはルールベース分析）に向けた前提条件を与えやすくなる。

### 1-C. フィルタ・ソート・検索機能の充実
- **課題**: プロジェクト数が増えた際に見づらくなる。
- **提案**:
  - インパクトやQWI順での**ソート機能**
  - 「進捗100%のものを隠す」「特定スコア以上のものだけ表示する」**フィルタ機能**
  - **テキスト検索バー**の追加
- **メリット**: 案件が増えてもダッシュボードとしての役割を維持できる。

## 2. 行動科学・フレームワークの深化

### 2-A. 重み付け（Weighting）のカスタマイズ機能
- **課題**: 現在の総合スコア計算 `QWI × 0.5 + LII × 0.3 + MSI × 0.2` は固定されている。
- **提案**: スライダー等でユーザー自身が「今は短期集中（QWI重視）」「今は長期資産づくり（LII重視）」のように数値を調整できるようにする。
- **メリット**: ユーザーの現在のライフフェーズに合わせた柔軟な意思決定支援が可能になる。

### 2-B. タイムライン（履歴）の記録と進捗グラフ
- **課題**: 「案件ごとの進捗バー」はあるが、過去の推移（いつスコアが上がったか）が追えない。
- **提案**: スコアや進捗の変更をローカルに履歴として保存し、折れ線グラフで過去1ヶ月間の成長を可視化する。
- **メリット**: ゲーミフィケーション要素が加わり、継続するモチベーション（Childlikeの維持）に繋がる。

## 3. レポートと出力の強化

### 3-A. ローカルLLM（WebGPU）との統合
- **課題**: 現状はGemini APIを排除したため、アドバイスが固定のルールベース生成になっている。
- **提案**: WebGPUを活用したブラウザ内LLM（例: WebLLM）を導入し、完全オフラインでありながら高度な自然言語処理による個別アドバイスを復活させる。
- **メリット**: APIコストゼロ・サーバーレス・プライバシー完全保護の高度な分析AIを実現できる。

### 3-B. PDFエクスポート・印刷最適化
- **課題**: 現在の出力はMarkdown、CSV、JSONのみ。
- **提案**: `react-to-print` や `pdfmake` を使い、ダッシュボードのグラフを含む状態で綺麗なPDFとして出力できるようにする。
- **メリット**: 報告書としてのフォーマットが整い、紙に印刷して壁に貼るなどの用途に対応できる。
